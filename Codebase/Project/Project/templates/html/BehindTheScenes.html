Behind The Scenes

Behind The Scenes

How the project is built, our data workflow, and the technology powering the platform.

Data Sources

We collected satellite data using Google Earth Engine (GEE) and NASA datasets. The main mapped variables are:

  • NDWI (water presence) – surface water and drainage changes
  • Snow/Ice change (6-year trend) – long-term warming indicator
  • Slope – identifies terrain more prone to erosion
  • Vegetation index – soil stability from plant cover
  • Ground temperature – detects permafrost thaw

Model Processing

All variables were processed in Python and scaled from 1 to 5, where:

5 = highest thaw / erosion risk

Ground temperature is the most important factor because thawed permafrost loses strength and erodes faster.

Permafrost Detection

  • Permafrost = ground frozen for at least two years
  • Above 0 °C = active thaw
  • Thaw weakens soil and increases erosion

Erosion Risk Logic

  • Slope – steeper land fails more easily
  • Water (NDWI) – runoff accelerates ground loss
  • Vegetation – reduced cover = weaker soil
  • Snow/Ice decline – long-term warming trend
  • Temperature – thawed ground is unstable

Final Index

All variables are combined into a single Erosion Risk Index.

The higher the index → the more likely the land is actively thawing and eroding.

Map Visualization

We built an interactive map using an open-source mapping platform combined with OpenWeather API data to visualize erosion risk across Canada.

The frontend is powered by JavaScript, which renders a color-coded risk layer so users can scan regions and instantly see which areas face low, medium, or high erosion risk.

System Overview

Backend (Data Processing)

  • Python handles all data retrieval, filtering, and calculations.
  • Core variables calculated: ground temperature, slope, vegetation, NDWI, and snow/ice change.
  • Each factor was normalized and scored from 1 to 5 based on its impact on permafrost thaw and erosion.
  • Scores were combined using a weighted formula to generate the final Erosion Risk Index.

Frontend (Visualization)

  • JavaScript displays the processed data on an interactive map of Canada.
  • The map uses color-coded markers to show erosion severity levels.
  • Users can visually compare regions and quickly identify high-risk areas.
  • This makes complex geospatial data understandable at a glance.
